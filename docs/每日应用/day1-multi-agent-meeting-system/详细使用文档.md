# Day 1 详细使用文档

本文档面向第一次接触本仓库的开发者，目标是帮助你在最短时间内把 Day 1 项目跑起来，并理解它的使用方式、输出内容和常见问题。

## 1. 项目简介

Day 1 项目是一个命令行版的会议纪要与行动项追踪系统。它会完成下面这条链路：

1. 读取本地音频文件
2. 使用 Whisper 做语音转写
3. 从转写文本里提取行动项
4. 把会议和行动项写入 SQLite 数据库
5. 通过命令行查看会议列表、行动项和摘要

当前项目更适合：

- 学习 Whisper + SQLite + CLI 的组合方式
- 搭建会议纪要类产品的 PoC
- 给后续 Web 界面或自动化流程做底层原型

当前项目不适合直接当成熟商用产品使用，因为行动项提取仍以规则法为主，示例音频也比较简单。

## 2. 仓库位置与入口

仓库根目录提供了统一入口：

```bash
python run_day1.py --help
```

核心代码位于：

```text
src/每日应用/day1-multi-agent-meeting-system/
```

推荐优先使用根目录入口 `run_day1.py`，这样命令路径更稳定，也更符合 README 的说明。

## 3. 运行前提

开始之前，请先确认下面几个前提：

- Python 版本：推荐 `3.10`、`3.11` 或 `3.12`
- 已安装 `ffmpeg`
- 首次执行 `upload` 时允许联网下载 Whisper 模型

不推荐：

- Python `3.13+`

原因是 `openai-whisper` 在较新的 Python 版本上可能存在兼容性问题，容易出现安装失败或构建失败。

## 4. 第一次安装

### 4.1 Clone 仓库

```bash
git clone https://github.com/sawyerxy/ai-daily-challenges.git
cd ai-daily-challenges
```

### 4.2 创建虚拟环境

Windows:

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

macOS / Linux:

```bash
python3.10 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 4.3 检查 FFmpeg

```bash
ffmpeg -version
```

如果这个命令能打印版本号，说明 FFmpeg 已经就绪。

## 5. 快速验证是否安装成功

最简单的检查方式有两个。

### 方式一：查看帮助

```bash
python run_day1.py --help
```

如果能看到 `upload`、`list`、`actions`、`summary`、`test` 等命令，说明 CLI 入口正常。

### 方式二：运行内置自检

```bash
python run_day1.py test
```

它会验证：

- 行动项提取模块是否正常
- SQLite 读写是否正常
- 命令行路径是否正常

如果你还想把示例音频上传链路一起测掉，可以执行：

```bash
python run_day1.py test --include-audio
```

这个命令会更慢，因为它会实际调用 Whisper。

## 6. 最常用的命令

### 6.1 上传音频并处理

```bash
python run_day1.py upload path/to/meeting.mp3 --title "项目启动会"
```

说明：

- `path/to/meeting.mp3` 是你自己的音频文件路径
- `--title` 可选，不写时默认使用音频文件名作为标题

成功后你会看到类似输出：

```text
正在处理音频文件: ...
会议标题: 项目启动会
正在加载Whisper tiny模型...
模型加载完成
正在转录音频: ...
会议记录已保存，ID: 1
转写文本长度: 1234 字符
提取到 2 个行动项:
...
```

### 6.2 查看会议列表

```bash
python run_day1.py list
```

这个命令会列出最近的会议记录，包括：

- 会议 ID
- 标题
- 日期
- 时长
- 行动项数量

### 6.3 查看行动项

查看全部行动项：

```bash
python run_day1.py actions
```

只看某个会议的行动项：

```bash
python run_day1.py actions --meeting-id 1
```

只看某个状态的行动项：

```bash
python run_day1.py actions --status pending
```

### 6.4 查看会议摘要

```bash
python run_day1.py summary --meeting-id 1
```

这个命令会显示：

- 会议标题
- 日期
- 音频路径
- 创建时间
- 转写文本预览
- 对应的行动项列表

## 7. 用示例音频跑完整流程

仓库里已经带了一个示例音频：

```text
src/每日应用/day1-multi-agent-meeting-system/example_meeting.mp3
```

可以直接这样跑：

```bash
python run_day1.py upload src/每日应用/day1-multi-agent-meeting-system/example_meeting.mp3 --title "示例会议"
python run_day1.py list
python run_day1.py summary --meeting-id 1
```

注意：

- 这个示例音频主要用于验证流程通不通
- 它并不是高质量会议录音
- 所以转写和行动项效果不一定好看

## 8. 数据库文件保存在哪里

默认情况下，数据库写入到这里：

```text
src/每日应用/day1-multi-agent-meeting-system/data/meetings.db
```

这意味着：

- 数据库不会乱写到你当前终端目录
- 只要项目目录还在，数据就还在
- 别人 clone 仓库时不会带上你的本地运行数据

## 9. 自定义数据库路径

如果你希望把数据单独放到别的目录，可以这样指定：

```bash
python run_day1.py --db-path ./temp/my_meetings.db list
python run_day1.py --db-path ./temp/my_meetings.db upload demo.mp3 --title "测试会议"
```

也可以使用环境变量：

Windows PowerShell:

```powershell
$env:MEETING_DB_PATH="C:\data\meetings.db"
python run_day1.py list
```

macOS / Linux:

```bash
export MEETING_DB_PATH=./temp/meetings.db
python run_day1.py list
```

## 10. 选择 Whisper 模型大小

默认模型是：

```text
tiny
```

如果你想调高精度，可以切换：

```bash
python run_day1.py --model-size base upload meeting.mp3 --title "项目周会"
python run_day1.py --model-size small upload meeting.mp3 --title "项目周会"
```

大致选择建议：

- `tiny`: 最快，适合快速验证
- `base`: 更平衡
- `small`: 更慢但通常更准
- `medium` / `large`: 对机器资源要求更高

也可以通过环境变量设置默认模型：

Windows PowerShell:

```powershell
$env:MEETING_MODEL_SIZE="base"
python run_day1.py upload meeting.mp3 --title "项目周会"
```

macOS / Linux:

```bash
export MEETING_MODEL_SIZE=base
python run_day1.py upload meeting.mp3 --title "项目周会"
```

## 11. 项目输出了什么

每次上传音频后，项目会产出三类结果：

### 11.1 终端输出

你会在终端看到：

- 当前处理进度
- 是否提取到行动项
- 会议 ID
- 摘要预览

### 11.2 SQLite 数据

数据库里有两张表：

- `meetings`
- `action_items`

`meetings` 用来存会议本体，`action_items` 用来存提取出的待办事项。

### 11.3 Whisper 模型缓存

第一次运行时，Whisper 会下载模型文件到本机缓存目录。这个过程是正常的，不属于项目 bug。

## 13. 常见问题排查

### 13.1 `pip install -r requirements.txt` 失败

优先检查：

- 你是不是用的 Python `3.13+`
- 是否已经创建并激活了虚拟环境
- `pip` 是否太旧

建议顺序：

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 13.2 `未安装 openai-whisper`

说明语音依赖没有装好。先确认：

- 当前终端是否进入了项目虚拟环境
- 依赖是不是安装到了别的 Python 版本

### 13.3 `ffmpeg` 找不到

说明系统 PATH 里没有 FFmpeg。先安装 FFmpeg，再重开一个终端验证：

```bash
ffmpeg -version
```

### 13.4 `list`、`actions`、`summary` 为什么不需要 Whisper 也能跑

这是项目现在刻意做的稳定性优化。只有在执行 `upload` 时才会真正加载 Whisper，这样日常查看数据不会被语音依赖拖住。

### 13.5 示例音频识别结果为什么很短

示例音频的价值主要是“验证流程”，不是“展示高质量识别效果”。如果你想看更真实的结果，请换成一段你自己的中文会议录音。

## 14. 开发者视角：如果你要继续扩展

建议从这几个文件入手：

- `run_day1.py`
  作用：仓库根目录统一入口
- `src/每日应用/day1-multi-agent-meeting-system/cli.py`
  作用：命令行参数和主流程编排
- `src/每日应用/day1-multi-agent-meeting-system/audio_processor.py`
  作用：Whisper 转写和音频时长读取
- `src/每日应用/day1-multi-agent-meeting-system/action_extractor.py`
  作用：行动项提取规则
- `src/每日应用/day1-multi-agent-meeting-system/database.py`
  作用：SQLite 读写

如果你要提升效果，最值得做的几件事通常是：

- 把规则法行动项提取升级成 LLM 或更强的 NLP 方案
- 增加 Web UI
- 增加状态更新、负责人筛选、导出功能
- 增加自动化测试和 CI

## 15. 推荐的最短上手路径

2. 用 Python `3.10` 创建虚拟环境
